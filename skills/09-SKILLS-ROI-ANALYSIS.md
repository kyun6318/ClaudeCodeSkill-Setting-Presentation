# Skills ROI Analysis — 스킬별 투자 대비 효과 분석

## 분석 프레임워크

모든 스킬의 ROI를 3가지 축으로 측정:

| 축 | 측정 항목 |
|---|----------|
| **투자 (Cost)** | 스킬 실행에 소비되는 토큰, 시간, 비용 |
| **수익 (Return)** | 절감되는 토큰, 줄어드는 재작업, 높아지는 품질 |
| **순효과 (Net ROI)** | 수익 - 투자. 양수면 가치 있음 |

> 아래 수치는 SKILL.md에 명시된 수치 + 파이프라인 구조에서 추정한 값이다.
> 실측 데이터가 아닌 **설계 기반 추정치**임을 전제한다.

## 1. 토큰 경제학 — 직접 절감

### 1-1. CLI 출력 압축 (token-compressor)

개별 명령어별 절감률:

| 명령어 | 변환 전 | 변환 후 | 절감률 |
|--------|--------|--------|-------|
| `git diff` | 전체 패치 | `git diff --stat` | **80%** |
| `git status` | 상세 출력 | `git status --short` | **85%** |
| `git log` | 전체 로그 | `git log --oneline -10` | **90%** |
| `pytest` | 전체 출력 | `pytest --tb=short -q` | **94%** |
| `npm test` | 전체 출력 | `npm test 2>&1 \| tail -20` | **90%** |
| `npm run build` | 전체 출력 | `tail -20` | **85%** |
| `tsc` | 전체 출력 | 압축 | **80%** |
| `eslint` | 전체 출력 | 압축 | **75%** |
| `docker logs` | 전체 출력 | `tail -20` | **80%** |

**복합 절감 효과**:

```
Layer 1: 명령어 압축           → 60-90% 절감
Layer 2: 에이전트 도구 제한     → 추가 10-20% (불필요한 Write/Edit 차단)
Layer 3: RTK 프록시 (선택적)   → 추가 10-20%
─────────────────────────────────────────
합산: 80-95% 토큰 절감
```

**시나리오 계산**:

| pytest 전체 실행 1회 | ~5,000 | ~300 | 94% |

### 1-2. 모델 라우팅 비용 절감 (cost-model-router)

**Tier별 비용 구조**:

| Tier | 모델 | 상대 비용 | 적용 태스크 | 전체 비중 |
|------|------|----------|-----------|----------|
| FREE | - | $0 | 파일 읽기, 검색, 분석 | ~33% |
| CHEAP | Haiku | 1x | 단순 편집, 포맷팅, 주석 | ~47% |
| EXPENSIVE | Sonnet/Opus | 5-15x | 복잡한 로직, 아키텍처, 디버깅 | ~20% |

**구체적 시나리오 (15개 태스크 프로젝트)**:

```
라우팅 없이 (전부 Opus):
  15 tasks × $0.67/task = $10.00

라우팅 적용:
  5 tasks × $0.00 (FREE)  = $0.00
  7 tasks × $0.07 (CHEAP)  = $0.50
  3 tasks × $1.33 (EXPENSIVE) = $4.00
  ─────────────────────────────────
  합계: $4.50  (55% 절감)
```

**절감 범위**: 태스크 분포에 따라 **40-70%** 비용 절감

**ROI 계산**:

```

투자: 태스크당 +1,000 토큰
수익: 재작업 회피 시 -5,000~10,000 토큰
순ROI: +4,000~9,000 토큰 (재작업 발생 시)
손익분기: 5건 중 1건이라도 누락 방지하면 흑자

## 2. 시간 경제학 — 기획 단계

### 2-1. 기획 방법별 시간 투자 vs 재작업 감소

| 방법 | 투자 시간 | 재작업 확률 | 재작업 시 추가 시간 | 기대 총시간 |
|------|----------|-----------|-------------------|-----------|
| 기획 없이 바로 코딩 | 0분 | ~70% | 3-5시간 | **2.1-3.5시간** |
| /auto-planner (빠른 기획) | 5분 | ~40% | 2-3시간 | **0.9-1.3시간** |
| /requirements-dialogue (심층 기획) | 30-60분 | ~15% | 1-2시간 | **0.7-1.3시간** |
| /requirements-dialogue-anchored | 30-60분 | ~10% | 0.5-1시간 | **0.6-0.7시간** |
| /full-pipeline-orchestrator-pipeline | 60-90분 | ~5% | 0.5시간 | **1.1-1.5시간 + 높은 품질** |

**핵심 통찰**: 기획에 30-60분 투자하면 재작업 확률이 70% → 15%로 감소.
재작업 1회 비용(3-5시간)을 감안하면 **기획 투자 ROI는 300-500%**.

```
시나리오: 10개 기능 프로젝트

기획 없이:
  10 기능 × 70% 재작업 = 7건 × 3시간 = 21시간 재작업
  총: 10시간(코딩) + 21시간(재작업) = 31시간

/requirements-dialogue 사용:
  기획: 1시간
  10 기능 × 15% 재작업 = 1.5건 × 1.5시간 = 2.25시간 재작업
  총: 1시간(기획) + 10시간(코딩) + 2.25시간(재작업) = 13.25시간

절감: 31 - 13.25 = 17.75시간 (57% 절감)
투자: 1시간 기획
ROI: 17.75x
```

## 3. 품질 경제학 — 검증 단계

### 3-1. 품질 체인의 버그 탈출 방지 효과

각 검증 단계가 통과시키는 결함의 비율 (추정):

```
코드 생성 직후: 결함 100건 존재 (가정)
  │
  ├─ /verification (빌드/테스트)                     → 30건 차단  (70건 남음)
  ├─ /quality-gate (메트릭 게이트)                     → 15건 차단  (55건 남음)
  ├─ /code-reviewer (의미론적 분석)                    → 20건 차단  (35건 남음)
  └─ /powerqa (자동 수정 루프)                       → 25건 수정  (10건 남음)
  
  ※ /sync 등은 자동 체인이 아닌 수동 검증
  최종 탈출 결함: ~10건 / 100건 = 90% 포착률
```

**품질 체인 없이**: 결함 포착률 ~30-40% (테스트만 실행하는 경우)
**품질 체인 사용**: 결함 포착률 ~90%

**후반 결함 발견 비용 (업계 통계 기반 추정)**:

| 발견 시점 | 수정 비용 (상대값) |
|----------|------------------|
| 기획 단계 | 1x |
| 코딩 단계 | 5x |
| 테스트 단계 | 15x |
| 배포 후 | 100x |

**ROI 계산**:

```
품질 체인 투자: 태스크당 ~8,000 토큰 (verification+quality-gate+code-reviewer)
품질 체인 수익: 결함 1건 조기 발견 = 테스트 단계 수정 회피 → 15x → ~120,000 토큰 절감

손익분기: 15건 중 1건만 조기 발견해도 흑자
실제 포착률: ~60건/100건 (60%) → 대규모 흑자
```

## 9. 결론

### 숫자로 보는 51개 스킬의 총 가치

| 재작업 건수 | 7건 / 프로젝트 | 0.5건 / 프로젝트 | **-93%** |
| 결함 포착률 | 35% | 90% | **+157%** |

### 스킬 도입 우선순위 (효과 순)

```
[Must Have]  token-compressor → cost-model-router → pre-completion-verifier
[Should Have] requirements-dialogue → dependency-auto-executor → 품질 체인 절차 → kongkong2
[Nice to Have] autonomous-build-harness → self-improvement-loop → full-pipeline-orchestrator-pipeline
[Situational] 나머지 전체

> **51개 스킬의 핵심 가치는 "토큰 절감"이 아니라 "재작업 제거"에 있다.**

> 기획에 1시간 투자하여 21시간의 재작업을 2.25시간으로 줄이는 것 —
> 이것이 이 스킬 생태계의 본질적 ROI다.
