## 2. 청중 프로파일

- **규모**: 20명 이내, 전원 사석에서 1회 이상 안면 있음 → 세미 캐주얼 톤 허용
- **직군**: 백엔드 / 프론트엔드 / 시스템 엔지니어 (대략 10년차 현업)
- **도메인**: IT 업계 공통, 소속 도메인은 다양 → 특정 산업 전제 불가
- **AI/LLM 이해도**: 기초 개념은 전제 가능. 사용자~탐색자 편차 존재
- **Claude Code Skill**: "어떤 것인지 아는" 수준 전제 → 개념 설명은 짧게, 운영 관점에 시간 배분
- **관심 포인트**: Claude Code 도입으로 인한 업무 방식의 변화 (발표자는 그중 **자신의 사용 방법과 이유**에 집중)

### 건너뛰는 내용
- LLM/프롬프트 기초 개념, ChatGPT/Claude 앱 기본 사용법, AI 도입 당위성 설득

## 3. 발표 목표

### 3.1 핵심 목표
발표자가 현재 사용하는 **Claude Code Skills 운영 방식과 그 이유**를 공유한다.
(개별 스킬 전량 공개가 아니라 **방향성·방법론** 중심 + 일부 대표 스킬 예시)

### 3.2 Key Takeaway — 처음과 끝에 반복
1. 에이전트는 사람을 대체하는 도구가 아니다
2. 잘 쓰려면 **운영 원칙**이 필요하다
3. 스킬은 그 운영 원칙을 **반복 가능하게 고정**하는 장치다

> 한 줄 프레이밍: "Agentic Coding에는 정답이 없지만, 이런 **하나의 방향성**은 가능하다."

### 3.3 Call to Action
청중이 직접 사용해 보고 **각자의 경험·방법을 되공유**하도록 유도.

## 4. 톤 & 스타일

- **톤**: 세미 캐주얼 (친한 사람 간 기술 공유), 단정적 "정답" 어조 지양
- **1인칭 허용**: "내가 이렇게 쓴다"는 경험 중심 서술 권장
- **정밀 수치 자제**: 98.5%, 100% 같은 수치는 구두로 말하지 않고 "대부분", "주요" 수준으로 표현 (백업 슬라이드로만 보유)

## 5. 콘텐츠 공개 범위 (중요)

### 5.1 공개 — 구조·원칙·슬라이드 내 스니펫까지만
- ✅ 디렉토리 구조(`~/.claude/skills/`) 골격 — 슬라이드 캡처 형태
- ✅ SKILL.md **포맷/스키마** 예시 (프롬프트 본문은 축약)
- ✅ MCP 설정(`settings.json`의 `mcpServers`) **구조와 선택 이유**
- ✅ 대표 스킬 1~2개의 **포맷 예시** (내부 로직 전체 노출은 ❌)
- ✅ 분리 원칙, 세션/토큰 운영 패턴, 안티패턴 TOP 3
- ✅ 전체 스킬 체계의 **범주 맵** (이름 일부 노출 가능, 전체 나열은 ❌. 규모는 구두로 "약 65개 수준"만 언급)

### 5.2 비공개
- ❌ **외부 저장소 공개 없음** (GitHub/Gist 링크 포함 X)
- ❌ 사용하는 전체 스킬 리스트와 내부 프롬프트 본문
- ❌ 개인 워크플로우 전용 커스텀 스킬 상세
- 📌 원칙: **"방향성과 방법 + 슬라이드에 담을 수 있는 최소 예시"만 제공, 외부에 가져갈 수 있는 자료는 제공하지 않는다**
